## 一、技术架构总览

```
浏览器 (Vue3 + Element Plus)
        │  Axios / RESTful JSON
        ▼
Spring Boot  Controller → Service → Mapper(MyBatis Plus) → MySQL
        │                              │
        └─ JWT 拦截器、全局异常、定时任务、本地文件上传
```

| 技术 | 用途 |
|------|------|
| Spring Boot 3.2 | IoC/DI、Web、定时任务、自动配置 |
| MyBatis Plus | ORM、分页、BaseMapper CRUD |
| MySQL | 业务数据持久化 |
| JWT (jjwt) | 无状态登录鉴权 |
| EasyCaptcha | 图片验证码 |
| BCrypt | 密码加密 |
| Vue 3 + Vite | 单页面前端 |
| Element Plus | UI 组件 |
| Pinia | 登录态 |
| Axios | HTTP 请求 |

## 二、后端功能清单与实现

### 2.1 用户注册与登录

| 功能点 | 实现方式 | 相关类/接口 |
|--------|----------|-------------|
| 图片验证码 | `SpecCaptcha` 生成 Base64 图，UUID 为 key 存内存 `CaptchaStore` | `GET /api/auth/captcha` |
| 普通用户注册 | 字段校验后写入 `sys_user`，状态 `PENDING` | `POST /api/auth/register` |
| 商家注册 | 额外写入 `merchant_info`（执照、身份证图、店铺名） | `POST /api/auth/register/merchant` |
| 管理员审核后登录 | 登录校验 `status=APPROVED`，否则拒绝 | `AuthService.login` |
| JWT 令牌 | 登录成功返回 token，拦截器解析写入 `UserContext` | `JwtUtil`、`AuthInterceptor` |

**用户信息字段**：id、姓名(realName)、手机、邮箱、城市、性别、银行账号(16位)、角色、状态。

### 2.2 角色与权限

| 角色 | 能力 |
|------|------|
| USER | 浏览、搜索、购物车、下单、钱包/积分、退货、评价 |
| MERCHANT | 发布/下架商品、店铺展示、处理退货、评价买家 |
| ADMIN | 审核用户/商品、用户 CRUD、充值、商家等级、封禁、拉黑、轮播 |

权限：`AuthInterceptor` + 业务层 `UserContext.getRole()` 校验。

### 2.3 商品管理

| 功能点 | 实现 |
|--------|------|
| 发布商品(多图) | `ProductService.publish` + `product_image` 表 |
| 商品字段 | 名称、类别、原价/售价、尺寸、成色、描述、议价、库存 |
| 管理员审核 | 状态 `PENDING` → `PUBLISHED` / `REJECTED` |
| 商家下架 | 状态 → `OFF_SHELF` |
| 店铺维度展示 | `GET /api/shops/{merchantId}` 含销量、好评率 |
| 搜索排序 | 名称模糊查询；按价格/销量/好评率排序 | `ProductService.search` |

### 2.4 购物车与订单

| 功能点 | 实现 |
|--------|------|
| 购物车多商家 | `cart_item` 按用户+商品唯一 |
| 一键下单 | `OrderService.checkout` 支持 cartItemIds 批量 |
| 钱包扣款(模拟支付) | `WalletService.pay` 扣余额，无第三方支付 |
| 线下交易信息 | 订单字段 `meetTime`、`meetPlace` |
| 商品锁定 | 下单后商品状态 `LOCKED`，减库存 |
| 确认收货 | 买家确认 → 结算货款给卖家 |
| 7天自动确认 | `@Scheduled` 定时任务 `OrderScheduledTask` |
| 资金托管 | 支付进入 `platform_account.escrow_balance` |
| 商家费率 | 按等级 0.1%~1% 扣平台费后结算卖家 | `Constants.MERCHANT_FEE_RATE` |
| 24小时退货 | `receiveTime + 24h` 校验；商家 `approveReturn` |
| 黑名单拦截 | `buyer_blacklist` 下单前检查 |

### 2.5 钱包与积分

| 功能点 | 实现 |
|--------|------|
| 用户钱包 | `wallet` 表 + 流水 `wallet_transaction` |
| 管理员充值 | `AdminService.rechargeWallet` |
| 消费得积分 | 确认收货后 1 元 = 1 积分 |
| 积分抵扣 | 100 积分 = 1 元，`PointsService.redeem` |

### 2.6 评价系统

| 功能点 | 实现 |
|--------|------|
| 商品评价(文字+五星) | `product_review`，更新商品 `avg_rating` |
| 买家评商家服务 | `buyer_merchant_review`，更新商家 `satisfaction_rate` |
| 商家评买家 | `merchant_buyer_review` |

### 2.7 可选功能

| 功能点 | 实现 |
|--------|------|
| 首页轮播 | `banner` 表 + `BannerController` |
| 商家限时封禁/关店 | `merchant_ban`，`PUBLISH_BAN` / `SHOP_CLOSE` |
| 恶意买家拉黑 | `buyer_blacklist`，merchantId 为空表示全平台 |

## 七、核心业务流程（下单→结算）

1. 用户加入购物车 → `CartService.add`
2. 一键下单 → 校验黑名单、库存、商品状态
3. 积分抵扣 → 扣减 `user_points`
4. 钱包扣款 → `wallet.balance` 减少
5. 商品锁定、生成订单明细，货款记入平台托管账户
6. 买家确认收货（或 7 天后定时任务）→ 按商家等级扣费率 → 余款打入卖家钱包 → 买家获得积分

## 九、测试建议（对应实验 6.2）

1. 注册普通用户 → 管理员审核 → 登录
2. 商家注册上传证照 → 审核 → 发布商品 → 管理员审核商品
3. 用户充值(管理员) → 加购 → 购物车一键下单 → 查看钱包扣款
4. 确认收货 → 查看卖家钱包与积分增加
5. 收货 24 小时内申请退货 → 商家同意/拒绝
6. 修改商家等级 → 下单验证不同费率
