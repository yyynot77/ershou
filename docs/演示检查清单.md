## 二、已知注意点（演示前规避）

1. **数据库**：必须执行 `sql/init.sql`，库名与 `application.yml` 一致（当前 `2023011345`）。
2. **审核**：新注册用户/商家、新商品默认 **待审核**，演示前用 `admin/admin123` 先通过，否则会登录失败或搜不到商品。
3. **买家余额**：下单前管理员给买家 **充值**，否则提示钱包不足。
4. **退货时限**：确认收货后 **24 小时内** 才能申请退货。
5. **发货顺序**：建议按 `已付款 → 发货 → 确认收货`，与检查清单一致。

## 三、推荐演示顺序（约 8～12 分钟）

### 准备（检查前 5 分钟）

1. 启动 MySQL，确认已执行 `init.sql`
2. 启动后端（8080），访问 http://localhost:8080/api/health
3. 启动前端（5173）
4. 用管理员登录，预先通过商家账号、上架 1～2 个商品

### 正式演示

| 步骤 | 角色 | 操作 |
|------|------|------|
| 1 | 商家 | 商家注册（或说明已审核）→ 发布商品 → 管理员审核商品 |
| 2 | 买家 | 注册 → 管理员审核 → 登录 |
| 3 | 管理员 | 给买家钱包充值（如 500 元） |
| 4 | 买家 | 浏览商品 → 加购/下单 → 填写线下交易时间地点 |
| 5 | 商家 | 商家订单 → **确认发货** |
| 6 | 买家 | 我的订单 → **确认收货** |
| 7 | 商家 | 个人中心/钱包流水查看 **收款**（可选展示） |
| 8 | 买家 | 申请退货（24h 内） |
| 9 | 商家 | 同意退货 → 买家钱包退款 |
| 10 | 商家 | 商品管理 → 下架商品（收尾） |

## 四、技术问答参考（1～3 题）

| 可能问题 | 简要回答 |
|----------|----------|
| 用的什么技术栈？ | 后端 Spring Boot 3 + MyBatis Plus + MySQL；前端 Vue 3 + Element Plus + Axios；JWT 登录鉴权 |
| 前后端如何交互？ | RESTful API，JSON；前端 Axios 带 Token，后端拦截器解析 JWT |
| 订单状态怎么设计？ | PAID → SHIPPED → RECEIVED → COMPLETED；另有 RETURN_REQUEST、RETURNED |
| 钱怎么流转？ | 下单扣买家钱包 → 平台托管 → 确认收货结算给商家（扣等级费率） |
| 为什么用 MyBatis Plus？ | 简化 CRUD、分页，Mapper 继承 BaseMapper |
| 验证码怎么实现？ | 纯 JDK 画 PNG + 内存存 key，避免 JDK17 无 Nashorn 问题 |
